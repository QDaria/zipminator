## Overview

Zipminator v1.0.0-beta.1 is the first public beta of the world's first Post-Quantum Cryptography (PQC) super-app. It delivers 8 operational pillars of military-grade encryption infrastructure powered by ML-KEM-768 (NIST FIPS 203) with true quantum entropy from IBM Quantum (156 qubits).

This release implements NIST FIPS 203 (ML-KEM-768) for key encapsulation. It does NOT hold FIPS 140-3 certification (CMVP validation has not been pursued).

## What's Included

### Crypto Core (Rust, 302 tests passing)

- ML-KEM-768 key encapsulation (Kyber768): `keypair()`, `encapsulate()`, `decapsulate()`
- QRNG entropy pool aggregating from IBM Quantum, Rigetti, QBraid with OS fallback
- Post-Quantum Double Ratchet protocol (forward secrecy, AES-256-GCM payloads, HKDF-SHA-256 chain keys)
- PQ-SRTP key derivation for VoIP media protection
- PQ-WireGuard handshake for VPN tunnel establishment
- Email crypto library (ML-KEM-768 key exchange, AES-256-GCM at rest)
- PII scanner (20+ PII types, auto-detection before encryption)
- Constant-time operations verified via dudect tests
- NIST KAT test vectors passing

## Known Limitations

### Crypto and Security

- No FIPS 140-3 validation (CMVP certificate not pursued; estimated $80-150K)
- Kyber768 implementation is the Rust workspace crate, not a NIST-certified module
- VoIP media protection constrained by WebRTC's built-in SRTP (cannot replace DTLS-SRTP at browser level)
- Q-VPN packet wrapping has prototype shortcuts; production WireGuard integration incomplete

### Platform

- macOS DMG is ad-hoc signed (not Apple Developer notarized); users must run `xattr -cr` or copy to writable location
- iOS and Android builds are debug-only; no TestFlight or Play Store distribution yet
- No release APK with `--split-per-abi` (debug APK is 160 MB)
- Linux and Windows Flutter builds not yet tested
- ZipBrowser uses system WebView, not a custom browser engine

### Features

- Pillar 9 (Q-Mesh/RuView) is at 15% -- architecture only, no code integration
- No message persistence in PQC Messenger (in-session only)
- No SMTP/IMAP server deployed for Quantum Mail (crypto library ready)
- Q-AI Assistant requires user-provided model files for local LLM mode
- Self-destruct UI wiring incomplete in Tauri desktop
- Anonymizer L8-10 stubbed by design (requires Enterprise QRNG allocation)

### Infrastructure

- API requires PostgreSQL + Redis to start (not bundled)
- Python `.so` binding is platform-specific (macOS arm64 only in this build)
- `flutter analyze`: 0 issues, but Flutter web build is 36 MB (needs tree-shaking)

## Installation

### macOS (Tauri Browser -- ZipBrowser)

```bash
# 1. Download the DMG
# Location: target/release/bundle/dmg/Zipminator_0.2.0_aarch64.dmg (5.7 MB)

# 2. Mount and copy to Applications
hdiutil attach Zipminator_0.2.0_aarch64.dmg
cp -R /Volumes/Zipminator/Zipminator.app /Applications/
hdiutil detach /Volumes/Zipminator

# 3. Clear quarantine (ad-hoc signed, no notarization)
xattr -cr /Applications/Zipminator.app

# 4. Launch
open /Applications/Zipminator.app
```

**Requirements**: macOS 11.0+ (Big Sur), Apple Silicon (M1/M2/M3/M4/M5)

### Web

Visit [https://www.zipminator.zip](https://www.zipminator.zip) -- no installation required.

### Flutter Super-App (Development)

```bash
cd app
flutter pub get
flutter run  # launches on connected device/simulator
```

## Upgrade Path

This is the first beta release. There is no prior version to upgrade from.

### Migration Notes for Future Releases

- Kyber768 key format is stable (PK: 1184 bytes, SK: 2400 bytes, CT: 1088 bytes)
- Entropy pool format (`quantum_entropy_pool.bin`) is append-only and forward-compatible
- Flutter state management uses Riverpod 3 `Notifier<T>` pattern; no migration needed
- Database schema (PostgreSQL) is not yet finalized; expect migrations in future betas
