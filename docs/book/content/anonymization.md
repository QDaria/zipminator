# 10-Level Data Anonymization

Zipminator provides a progressive anonymization system with 10 levels, ranging from basic SHA-256 hashing to Paillier homomorphic encryption. Each level is designed for different use cases, regulatory requirements, and data sensitivity levels.

```{admonition} Pseudonymization vs. Anonymization (GDPR Art. 4(5))
:class: note

Levels 1-4 are **pseudonymization** techniques: the data can be re-identified using additional information (hash reversal via lookup, token maps, or deterministic mappings). These remain "personal data" under GDPR.

Levels 5-10 are **anonymization** techniques: the transformation is statistically irreversible. Properly anonymized data falls outside GDPR scope entirely.
```

## Overview

| Level | Technique | Category | Tier Required | Reversible? |
|------:|-----------|----------|:-------------:|:-----------:|
| 1 | SHA-256 hashing | Pseudonymization | Free | No* |
| 2 | Quantum-random replacement | Pseudonymization | Free | No |
| 3 | Deterministic tokenization | Pseudonymization | Free | Yes (with map) |
| 4 | Generalization (ranges/categories) | Pseudonymization | Developer | No |
| 5 | Suppression (NULL replacement) | Anonymization | Developer | No |
| 6 | Quantum noise injection | Anonymization | Pro | No |
| 7 | Synthetic data generation (Faker) | Anonymization | Pro | No |
| 8 | k-Anonymity grouping | Anonymization | Enterprise | No |
| 9 | Differential privacy (Laplace noise) | Anonymization | Enterprise | No |
| 10 | Paillier homomorphic encryption | Anonymization | Enterprise | Yes (with key) |

*L1 hashes are deterministic; identical inputs produce identical outputs, enabling re-identification via rainbow tables.

## Quick Start

```python
from zipminator.crypto.anonymization import AnonymizationEngine
import pandas as pd

engine = AnonymizationEngine()

df = pd.DataFrame({
    "name": ["Alice", "Bob", "Charlie", "Diana"],
    "salary": [55000, 62000, 71000, 48000],
    "email": ["alice@corp.com", "bob@corp.com", "charlie@corp.com", "diana@corp.com"],
})

# Apply level 1 (SHA-256 hashing) to the name column
result = engine.apply_anonymization(df, columns=["name"], level=1)
```

## Level-by-Level Reference

### Level 1: SHA-256 Hashing

One-way cryptographic hash. Identical inputs always produce the same hash, making it useful for deduplication without exposing the original value.

```python
result = engine.apply_anonymization(df, columns=["name"], level=1)
print(result["name"][0])
# "2bd806c97f0e00af1a1fc3328fa763a9269723c8db8fac4f93af71db186d6e90"
```

### Level 2: Quantum-Random Replacement

Replaces each value with a 10-character random string generated by the `QuantumRandom` engine. When quantum entropy is available, the randomness is seeded from IBM Quantum hardware.

```python
result = engine.apply_anonymization(df, columns=["name"], level=2)
print(result["name"][0])
# "X7K2M9QR4P" (random each time)
```

### Level 3: Deterministic Tokenization

Maps each unique value to a stable token based on its MD5 hash. The same input always produces the same token, preserving referential integrity across datasets. The token map is stored in `engine._token_maps` for optional reversal.

```python
result = engine.apply_anonymization(df, columns=["name"], level=3)
print(result["name"][0])
# "TOKEN_2BD806C9"
```

### Level 4: Generalization

Numeric values are bucketed into ranges (default bucket size: 10). Text values are reduced to their first-letter category.

```python
result = engine.apply_anonymization(df, columns=["salary"], level=4)
print(result["salary"][0])
# "55000-55010"

result = engine.apply_anonymization(df, columns=["name"], level=4)
print(result["name"][0])
# "CATEGORY_A"
```

### Level 5: Suppression

Replaces all values with `pd.NA` (NULL). The column structure is preserved but all data is removed. Useful for columns that must exist in the schema but cannot contain any values.

```python
result = engine.apply_anonymization(df, columns=["email"], level=5)
print(result["email"][0])
# <NA>
```

### Level 6: Quantum Noise Injection

For numeric columns, adds noise proportional to the value (10% noise factor) using quantum random numbers. For non-numeric columns, falls back to quantum-random replacement.

```python
result = engine.apply_anonymization(df, columns=["salary"], level=6)
print(result["salary"][0])
# 53724.81 (original 55000 +/- ~10%)
```

### Level 7: Synthetic Data Generation

Uses the Faker library to generate realistic synthetic data based on column name hints. Recognized column names include: name, email, phone, address, company, ssn, credit card.

```python
result = engine.apply_anonymization(df, columns=["name"], level=7)
print(result["name"][0])
# "Jennifer Martinez" (realistic but entirely synthetic)
```

Requires `pip install zipminator[anonymization]` for the Faker dependency. Without Faker, falls back to random string generation.

### Level 8: k-Anonymity Grouping

Generalizes quasi-identifiers so that every combination of values appears at least k times in the dataset (default k=5). Numeric columns are bucketed; categorical columns are grouped.

```python
result = engine.apply_anonymization(df, columns=["salary"], level=8)
# Salary values grouped into ranges ensuring k=5 equivalence classes
```

### Level 9: Differential Privacy (Laplace Noise)

Adds calibrated Laplace noise to numeric columns. The noise scale is determined by the data sensitivity (max - min) and the privacy budget (epsilon=1.0). For categorical data, applies randomized response with probability p=0.75.

```python
result = engine.apply_anonymization(df, columns=["salary"], level=9)
# Salary values have Laplace noise added (epsilon=1.0)
```

Requires numpy. The privacy guarantee is epsilon-differential privacy, meaning individual records cannot be identified with confidence above `e^epsilon`.

### Level 10: Paillier Homomorphic Encryption

Encrypts numeric values using the Paillier cryptosystem. The encrypted values support addition and scalar multiplication without decryption, enabling computation on encrypted data.

```python
result = engine.apply_anonymization(df, columns=["salary"], level=10)
# Salary values are Paillier-encrypted objects
# Compute on encrypted data:
# encrypted_sum = result["salary"][0] + result["salary"][1]
# decrypted = engine.private_key.decrypt(encrypted_sum)
```

Requires `pip install phe`. Without it, falls back to Level 9 (differential privacy). The Paillier keypair is generated on first use and stored in `engine.public_key` / `engine.private_key`.

## Subscription Gating

Levels 1-3 are always available without authentication. Levels 4-10 require an API key or activation code:

```python
from zipminator.crypto.subscription import APIKeyValidator

# Check if a level is accessible
allowed, msg, method = APIKeyValidator.authorize_level(3)
# allowed=True, method="free_tier"

allowed, msg, method = APIKeyValidator.authorize_level(7)
# allowed=False, msg="Level 7 requires Pro tier or higher..."

# Set API key for higher levels
import os
os.environ["ZIPMINATOR_API_KEY"] = "your-key"
allowed, msg, method = APIKeyValidator.authorize_level(7)
```

See {doc}`subscription` for full tier details and activation code formats.
